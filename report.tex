\section{Obiettivo del Progetto}

Creeping Crawler è una pipeline end-to-end per l'acquisizione e l'analisi di contenuti testuali da sorgenti web eterogenee, progettata per alimentare sistemi RAG e modelli linguistici. Il sistema scarica pagine web tramite Crawl4AI, le trasforma in testo strutturato tramite parser specifici per dominio, e ne valuta la qualità di estrazione confrontando l'output con un Gold Standard costruito manualmente.

% ─── 2. Organizzazione del codice ────────────────────────────────────────────

\section{Organizzazione del Codice}

Il progetto è suddiviso in due container Docker orchestrati da \texttt{docker-compose.yaml}: \textbf{backend} (FastAPI, porta 8003) e \textbf{frontend} (FastAPI + Jinja2, porta 8004).

\paragraph{Backend}

\begin{itemize}
  \item \textbf{\texttt{src/lib/crawling/}} — Modulo di acquisizione basato su Crawl4AI. \texttt{crawler.py} espone tre funzioni: \texttt{fetch\_page} (crawl live), \texttt{fetch\_page\_from\_html} (da HTML via prefisso \texttt{raw:}), e \texttt{fetch\_page\_for\_url} (usa l'HTML salvato nel Gold Standard come sorgente primaria, ricade sul crawl live solo se non disponibile). La sottocartella \texttt{domains/} contiene un \texttt{CrawlerRunConfig} per ciascun dominio con esclusione di tag e selettori CSS non pertinenti.

  \item \textbf{\texttt{src/lib/parsers/}} — Interfaccia astratta \texttt{ContentParser} con implementazioni per Wikipedia, ESPN, CNBC e XE. Ogni parser opera sul Markdown prodotto da Crawl4AI: interrompe la raccolta alle sezioni boilerplate e rimuove pattern residui tramite espressioni regolari.

  \item \textbf{\texttt{src/lib/evaluation/}} — Pipeline in tre moduli: \texttt{tokens.py} (stripping Markdown, normalizzazione Unicode), \texttt{token\_level.py} (precision, recall, F1), \texttt{similarity.py} (cosine, Jaccard, excess ratio).

  \item \textbf{\texttt{src/lib/gold\_standard/}} — Accesso ai file JSON in \texttt{gs\_data/}, uno per dominio, con le entry del Gold Standard.

  \item \textbf{\texttt{src/routes/}} — Quattro router FastAPI (\texttt{domains}, \texttt{parse}, \texttt{gold}, \texttt{evaluate}) con schemi Pydantic in \texttt{src/schemas/}.
\end{itemize}

\paragraph{Frontend}

Server FastAPI con rendering Jinja2. \textbf{\texttt{GET /}} espone il menù degli URL del Gold Standard raggruppati per dominio. \textbf{\texttt{GET /compare?url=}} chiama \texttt{/parse}, calcola le metriche se l'URL è nel Gold Standard, e mostra i pannelli di testo con diff interattivo Monaco Editor.

% ─── 3. Valutazione Parser ───────────────────────────────────────────────────

\section{Valutazione dei Parser}

\paragraph{Metodologia.} Ogni parser viene valutato su 10 URL per dominio tramite \texttt{GET /full\_gs\_eval}. Il testo estratto viene passato per \texttt{strip\_markdown} prima del confronto, in modo da normalizzare quote tipografiche, trattini Unicode ed entità HTML. Le metriche set-based operano su insiemi di token unici (case-sensitive, separazione per spazio); le metriche vettoriali (cosine, Jaccard, excess ratio) operano sui conteggi di frequenza tramite \texttt{Counter}.

Durante i test sul dominio CNBC compariranno pertanto dei \textbf{warning} che segnalano l'utilizzo del fallback GET.

% ─── 4. Bonus/Extra ──────────────────────────────────────────────────────────

\section*{Bonus/Extra}

\paragraph{Similarity Eval (metriche aggiuntive).} Oltre alla \texttt{token\_level\_eval} obbligatoria, è stato implementato un secondo gruppo di metriche su vettori di frequenza (\texttt{Counter}): \emph{cosine similarity} (somiglianza di distribuzione, alta anche con testo extra), \emph{Jaccard similarity} (penalizza sia token aggiuntivi che mancanti), ed \emph{excess ratio} (frazione di token estratti non coperti dal Gold Standard; lower is better). Completano la valutazione set-based rilevando contenuto ridondante non catturato da precision/recall.

\paragraph{Script \texttt{crawl\_gs.py} e \texttt{Makefile}.} Uno script autonomo scarica tutti gli URL del Gold Standard e salva l'output in \texttt{gs\_results/} (ignorata da git) in più formati (HTML, Markdown, testo stripped e parsato). L'\texttt{--update-json} aggiorna il campo \texttt{html\_text} nei JSON del Gold Standard per il re-parsing offline. Il Makefile automatizza creazione ambienti, installazione dipendenze, avvio server e crawling.

\paragraph{Monaco Diff Editor.} La pagina dei risultati integra Monaco Editor in modalità diff affiancato, mostrando le differenze tra testo estratto e Gold Standard e guidando il tuning iterativo dei parser.

\paragraph{Endpoint aggiuntivi.} \texttt{GET /gold\_standard\_urls} restituisce la lista completa degli URL del Gold Standard, usata dal frontend per il menù a tendina e per determinare se un URL ammette valutazione.
